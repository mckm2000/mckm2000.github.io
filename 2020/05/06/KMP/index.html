KMP算法个人理解

KMP算法用于字符串的匹配问题,相比于传统的暴力解决,巧妙地运用了一个数组next来加速字符串的匹配,而整个算法最核心的也是最难的就是这个next的计算.

next数组的计算

我们有两个字符串,一个是主串S,一个是模式串T,而next数组主要是对模式串T进行计算.

我们设模式串T为: A B C D A B D

T A B C D A B D …
J 0 1 2 3 4 5 6
next -1 0 0 0 0 1 2 0

对于next[J]的意义为:T[J]前next[J]个字符与从头开始数的前next[J]个字符相同.

举个两个栗子:

  1. 设J为5,即T[J]为B,那么T[4]= T[0] = A,所以next[5] = 1
  2. 设J为6,即T[J]为D,那么T[4~5] = T[0~1] = AB,所以next[6] = 2

next数组的原理

上一部分是关于如何计算next数组的.当我们知道了怎么算和next的意义后,我们再来分析一下next数组的原理,也就是为什么next数组要这样算.

我们设主串S为: A B C D A B A …

S A B C D A B A
T A B C D A B D
J 0 1 2 3 4 5 6 …

如果我们不会KMP的话,字符串匹配我们大多都是暴力来做,一般是两个for循环来遍历:

上面暴力的算法的时间复杂度在最坏的情况下是O(N*M)

而之所以这么复杂的原因主要还是需要不停的把主串上的指针回溯,而KMP算法的原理就是消除主串指针的回溯.

在上表中,我们能很清楚的看到除了T[6]以外,T[0~5] = S[0~5],这时我们需要移动模式串T从而和主串S继续对比.

而next解决的就是移动多少的问题,我们很明显可以看出只有当T[0~K] == S[J-K-1~J-1]时我们才能开始继续对比.

换种说法:因为在J之前的字符都是相同的,而我们要把模式串T平移到从J开始比对,那么在J之前的字符一定是比对过相同的,那么对于平移过后的模式串T来说,J之前的就是新的开头,所以要把开头与J前的字符串来比较,且为了不遗漏,开头要尽量长,即next[J]要尽量大.
