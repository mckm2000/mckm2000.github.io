湖南大学第十六届程序设计竞赛（重现赛）

1.暴力 数学 A.Triangles

题目大意

在平面直角坐标系上给你三个点$(x_i,y_i)\in [-10000,10000],i=1,2,3$，问三个点构成什么三角形，注意有可能不构成三角形，如三点共线。

题目思路

我直接暴力，先根据输入的三个点的坐标计算出三条直线的长度的平方即$d_1,d_2,d_3$，然后就开始套if了：

  1. 不构成三角形：当两条直线斜率相同，即$\frac{y_1-y_2}{x_1-x_2} = \frac{y_2-y_3}{x_2-x_3}$时三点共线，不过注意除法要特别考虑除数为0的情况，所以建议直接移项转换为乘法，即$(y_1-y_2)\times(x_2-x_3） = (y_2-y_3)\times(x_1-x_2)$；
  2. 直角三角形：根据勾股定理，即当出现$d_1+d_2=d_3$时三点构成直角三角形;
  3. 钝角三角形：直角时$d_1+d_2=d_3$,当其变为钝角时$d_1+d_2<d_3$；
  4. 排除掉上面几种情况后自然只剩下锐角三角形了

2.数学期望 D.Queuing

题目大意

张三准备去食堂吃饭，而在到了食堂后张三发现食堂有n个窗口，而张三是第m个到食堂的人，假设每个人到每个窗口的时间都是一样的，即到达食堂的先后顺序决定了他到达窗口的先后顺序，同时每个人选择任意一个窗口的概率都是$\frac{1}{n}$，求张三在队伍中排名的期望，精确到小数点后6位。

题目思路

首先张三前面是有$m-1$个人的，因为我们要求的是张三在队伍中排名的期望，那么我们只关心张三所在这个队伍中张三前面有多少个人，所以对于前面的$m-1$个人来说，可以看作只有两种情况：

  1. 和张三一个队伍，概率$p_1=\frac{1}{n}$；
  2. 和张三不在一个队伍，概率$p_2=1-p_1=\frac{n-1}{n}$；

由此可以看出这$m-1$个人排在张三前面的事件$X$满足二项分布，即$X \sim B(m-1,\frac{1}{n})$，由二项分布的期望公式我们可以得到期望$E(X)=np=(m-1)\times(\frac{1}{n})$，由于我们求的是张三在队伍中排名的期望，即：

$张三的排名=张三前面的人数+1，E(张三的排名)=E(张三前面的人数)+1=E(X)+1$

4.思维题 G.Binbin’s string

题目大意

给你一个字符串S和一个字符串T，你有两种操作：

  1. 从字符串S中任意一个位置为开始，删除任意多个连续字符；
  2. 从字符串S中任意一个位置为开始，插入任意多个连续字符；

问最少需要几次操作可以使得S变得和T一样。

题目思路

其实仔细思考一下就会发现只有三种情况：

  1. 0次操作，即本来就已经一样了无需操作；
  2. 1次操作，即只需要加入或删除一段就可使得他们变得一样；
  3. 2次操作，即直接将整个字符串删完然后把T插入进去。

所以关键是判断一次操作是否可行。

其实无论把插入和删除看成修改，则有三种情况：

  1. 修改在字符串头，如果这个修改是删除，那么字符串S应该倒过来的部分和T完全匹配，如果这个修改是插入，那么这整个字符串S应该倒过来和T部分匹配；
  2. 修改在字符串尾，如果这个修改是删除，那么字符串S应该倒过来的部分和T完全匹配，如果这个修改是插入，那么这整个字符串S应该倒过来和T部分匹配；
  3. 修改在字符串中，修改的部分应该将原来的字符串分成两部分，且这两部分应该与T相应的部分匹配，如果这两部分的拼起来刚好和T一样则只需要一次操作；

比赛时一上来本来看的是G题，然后看到大家都快速过了F我就知道那是签到题了，于是就跳过去写了F，一开始看到$2^{63}$的数据范围还以为能用long long，不行后还试了unsigned long long，信导计组白学系列，都不行后就想到int128，但是我只是知道，但是没有去学怎么用，也没留板子，所以就只能用高精度了。然后G题一开始还想复杂了，WA了一发后慢慢重新想，最后想着想着豁然开朗然后就过了。然后A题上手直接无脑暴力梭哈，然后一发入魂。D题数学题推出来是二项分布，然后刚考完概率论的人不会算二项分布期望(丢死个人)，最后百度现学然后才过的。然后就是卡了几百年的I题，WA了几百次之后终于放下转去做L，上手就是一个BFS，结果一直段错误，又WA了几百遍才发现，有个地方的-写成了+，然后数组越界了。I题比赛结束后看了别人代码才发现，二分边界设错了，不过算补题，晚点再更新。

6.二分 思维 I.Binbin and Balls

题目大意

给你一栋大楼的楼层数$Y$，给你两个完全一样的球，这两个球在一定高度以上的楼层往下丢就会碎，碎了就不能再用了。然后你可以通过这两个球找出那个最大的丢下去不会碎的楼层X，问最坏情况下找出X最少需要丢几次球。

题目思路

这题思维上比较有难度首先要分析丢球的策略，假设我们随机一个楼层$y_1$丢第一个球，有两种情况：

  1. 没碎，那么代表着$[1,y_1-1]$这个球都不会碎；

  2. 碎了，那么现在我们只有一个球了，为了确保我们能找到X，我们只能把球从小到大往上扔，不然万一你又随便扔一个位置$y_2$，他又碎了你就找不到那个X了。为了防止不明白我再举个例子：

    1. 一开始直接在5楼丢，碎了；
    2. 然后第二个球直接在3楼丢，又碎了；

    那么你怎么知道这个X是1还是2呢？

    所以为了保证能找到，当第一个球碎了的时候，我们必须要从小到大开始丢。

那么整个寻找的过程就分为了两步：

  1. 找到在楼层$y$丢下会碎所用的次数$k_1$；
  2. 从小到大依次丢球的所用的次数$k_2$。

$k_1$很明显是与将楼层分成几段有关，而$k_2$的大小是碎裂那一段的楼层数，举个例子：

我们一开始在10楼丢没碎，然后我们在20楼丢碎了，这时候我们剩下的球需要从小到大丢，但是因为我们上次在10楼丢没碎已经代表了$[1,10]$是不会碎的，那么这时候我们就应该从11楼开始往下丢一直试到19楼，因为根据最坏情况，所以我们一定是要试完的，那么这时候我们的次数$k = 2 + 9 = 11$。

由此可见$k_2$的大小是与我们分段时每段楼层的层数有关。

那么我们要如何合理分配楼层数才能使得次数最坏情况下最少呢？

假如我们一开始只分两段即$[1,98][99,100]$：

  1. 先在98丢，碎了，那么我们的次数$k=1+97=98$；
  2. 先在98丢，没碎，再在100丢，碎了，那么我们的次数$k=2+1=3$。

但是根据最坏情况，我们的结果就是98。

那如果我们还是分两段，但是变成$[1,97][98,100]$:

  1. 先在97丢，碎了，那么我们的次数$k=1+96=97$；
  2. 先在97丢，没碎，再在100丢，碎了，那么我们的次数$k=2+2=4$。

根据最坏情况，我们的结果就是97。

以此类推，我们不难看出一段的次数可以通过将该段的楼层数分配给其他段来减小，那么想要这么多段中的最大值最小，我们就需要将次数尽量平均的分配，也就是使得每一段的次数尽量相等。

假设我们将楼层分成$n$段分别为$[1,y_1][y_1+1,y2]…[y_{n-1}+1,y_n]$,那么每段的次数$k_i$为：

$k_1=1+y_1-1=y_1$，$k_2 = 2 + y_2 - 1 - (y_1+1) = 2+y_2-y_1……$

$k_n= n + y_n - 1 - (y_{n-1} +1) = n+y_n-y_{n-1}$

则最坏情况下的最小次数$k=max(k_1,k_2,…,k_n)$。

为了让每一段的次数$k$尽量相等，因为第二段比第一段多丢了一次第一个球，所以从小到大丢第二个球的次数就应该比第一段少一次，以此类推可得出$y_1 - 1 = y_2 - y_1 + 1 = y_3 - y_2 + 2 = … = y_n - y_{n-1} + n - 1$，可以看出每段的长度是个公差为1的等差数列。

根据等差数列求和公式：$\frac{a_1+a_n}{2}\times n$，带入求得$n$段区间的楼层数的和为 $\frac{1+n}{2} \times n$，因为整栋楼的层数为$Y$，所以$n$应该满足$\frac{1+n}{2} \times n \ge Y$，所以我们只要找出满足条件的最小的$n$就可以得到我们的答案，而因为一定有一个丢下去会碎的楼层，所以不等式的右边需要换成$Y-1$，因为当在$Y-1$楼丢下去如果没碎，代表在$Y$楼丢下去一定会碎，为了满足最小则$n$应该满足$\frac{1+n}{2} \times n \ge Y - 1$。

这时就只需要用二分找一下最小的$n$了，然后我是移项成乘法做的，不过要注意二分的右边界要初始化成$2e9$，我一开始是$Y$，但是$Y$万一到了$1e18$，二分时会出现爆long long的情况，而初始化成$1e9$的话又可能出现覆盖不到的情况，因为不等式的右边$(Y-1) \times 2 < 2e18$。

7.搜索 B.Yuki with emofunc and playf

题目大意

给你一个$n\in[1,1e6]$和$x\in[1,1e9]$，一开始$k=1$你可以进行两种操作：

  1. $k = k\times10$；
  2. $k=k-1+x$；

问最少通过几次操作可以使得$k$变成$n$的倍数。

题目思路

每次都有两种操作可以选，则然后根据最少特性可以看出是使用BFS(广度优先搜索)，$k$是$n$的倍数，而$n$最大能到$1e6$，如果只是单纯的暴力肯定会爆内存的，所以我们需要优化一下，因为$k$是$n$的倍数即$k \mod n=0$，那么根据同余定理每一次操作都可以对$n$取模，这样子我们其实最多只需要搜$n$个点就能找到答案，那么这就转化成用BFS求从点1到点0的最短路问题。

我还稍微优化了一下内存，常规做法应该是vis存储到达每一个点所需要的操纵数，我直接把vis变成了bool，只用来判断有没有到达过这个点，然后根据BFS是一层一层向外扩散，用last储存当前层还有几个节点没搜，用next代表下一层有几个节点没搜，用times代表当前节点是第几层。
